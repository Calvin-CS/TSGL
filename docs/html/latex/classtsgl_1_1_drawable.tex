\doxysubsection{Detailed Description}
A class for drawing objects onto a \mbox{\hyperlink{classtsgl_1_1_canvas}{Canvas}} or \mbox{\hyperlink{classtsgl_1_1_cartesian_canvas}{Cartesian\+Canvas}}. 

\begin{DoxyWarning}{Warning}
{\bfseries{{\itshape Though extending this class must be allowed due to the way the code is set up, attempting to do so could potentially mess up the internal GL calls the library uses. Proceed with great caution.}}}
\end{DoxyWarning}
\mbox{\hyperlink{classtsgl_1_1_drawable}{Drawable}} provides a base class for drawing 3D objects to a \mbox{\hyperlink{classtsgl_1_1_canvas}{Canvas}} or \mbox{\hyperlink{classtsgl_1_1_cartesian_canvas}{Cartesian\+Canvas}}. \begin{DoxyNote}{Note}
\mbox{\hyperlink{classtsgl_1_1_drawable}{Drawable}} is abstract, and must be extended by the user.
\end{DoxyNote}
{\ttfamily vertices} should be an array of floating point values in T\+S\+GL\textquotesingle{}s vertex format.

{\ttfamily numberofvertices} should be the actual integer number of vertices to be drawn (e.\+g., {\itshape 36} for a cube).

{\ttfamily geometry\+Type} should be one of GL\textquotesingle{}s primitive drawing modes. See \href{https://www.opengl.org/sdk/docs/man2/xhtml/glBegin.xml}{\texttt{ https\+://www.\+opengl.\+org/sdk/docs/man2/xhtml/gl\+Begin.\+xml}} for further information.

Theoretically, you could potentially extend the \mbox{\hyperlink{classtsgl_1_1_drawable}{Drawable}} class so that you can create another \mbox{\hyperlink{classtsgl_1_1_drawable}{Drawable}} class that suits your needs.

However, this is not recommended for normal use of the T\+S\+GL library.
